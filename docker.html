Definition: Docker is first of all an open source software that allows developers to containerize their large apps into tiny containers. Didn't understand ? Well, let's look at an example. Take amazon for example all it's web pages are dynamically rendered and available to users in matter of milliseconds . Surely there is no computer that servers front end and back end that fast to billions of users who visit it every day right ? Well docker solves this issue.

How ?
As I said it allows developers to convert their large apps into tiny containers. So if amazon has 10 Node Js backedn servers running then docker makes it possible for the application to run on 10 different containers while being on the same domain name. Each Node Js application is ran in one different container with it's own dependencies. So that way each container can be ran on different Virtual CPU. Just google how many servers amazon uses.
I have seen many people confuse this with the API's running on different computers with unique domain names. In Docker all the 10 API are ran in different container while being part of the same app. So it's basically dividing an app into ten pieces while all the pieces functioning as a unit. This is different from when you run API on different computers.

You will get a clear idea on this once we build our own Docker Node JS app.
