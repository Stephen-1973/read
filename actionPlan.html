Skills you will learn.

These are the following skills which you will be learning by building our first app.

Custom Event Bus - An event bus is a Api that allows our different Api's to interact without one Api directly interacting with other.If an API to directly communicate with other API then it will cause a lot of miscommunication. Why? Let's say you are working in amazon and suddenly a Node Cluster went offline because it couldn't handle so many requests. This often happens a lot in big tech companies that's the reason there are 'Stand By Nodes'. Now an API directly interacting with the API that just went down. Imagine the problems it could cause. The API which is directly interacting with the other API will also go down because it couldn't interact with the API and then many others API's which interact with these API's will go down and finally the whole website will be down. You can prevent this by writing an on catch error. But is that really enough though ? No Not at all. It's not enough. Communication between API's in microservices app is such a huge problem that Google had to invent NATS Streaming Server. We will learning that in our 2nd app.

Backend Architecture - In a microservice app architecture is everything. Imagine you are working for a Bank that needs Backend API for Net Banking. If you don't use the infamous architecture for it and just use reading and writing Bank Balance for each customer using Database, you would be in big trouble. Because Database aren't know for their availability. In a huge application clusters go down frequently and if you can't write the deposited amount into the storage before the certain customer withdraw's then his money is gone. Let's talk about a better example. A customer's bank account has been deposited money worth of a Million Dollars and as soon as it's deposited the customer wants to withdraw it and at the crucial moment our database got too many requests and has insane latency. Once our API sends the request to the Database we update our client whoever deposited the money as the money has been deposited but the customer when tries withdraw since our Database has latency and can't update his amount then he won't be able to withdraw the money. Imagine how many complaints would be filing up against the bank. Insane right ? how a small mistake could damage the reputation of the bank.

FrontEnd and BackEnd Communication - Many people who have experience in the field of Front End don't know how to deal with the communication with Backend and few don't know how to dynamically render the data from API in the Front End. This will be the core of learning concept for this app.
